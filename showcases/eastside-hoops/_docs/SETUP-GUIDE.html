How Data Flows

Architecture: Phil fills in the Fixtures tab → the Apps Script onEdit() trigger fires → Standings tab recalculates → the website fetches both tabs via the Google Visualization API endpoint (/gviz/tq?sheet=...&tqx=out:json) → data-manager.js parses the JSON and passes it to the render functions in app.js.
CORS note: The Google Visualization API endpoint has CORS restrictions that block requests from localhost. The site will show sample/fallback data during local development — this is expected behaviour. On a deployed domain (Netlify, GitHub Pages, etc.) it works correctly.

How to Add/Update Fixtures (Phil's job)

Adding a new fixture

Go to the Fixtures tab and add a new row. Phil selects the Team from a dropdown — no free-typing needed.

Leave Score and Result blank for upcoming games.

Recording a result

Find the fixture row and fill in Score (e.g. 52-48) and select Result from the dropdown (W / L / D). Standings recalculate automatically.

Score format: Vikings score first — 52-48 means Vikings 52, opponent 48.
Standings tab: Phil never needs to touch this. It is fully managed by the Apps Script. If standings look stale, Phil can use the 🏀 Vikings → Recalculate Standings menu option.

Adding a New Team or League

Teams are defined in the CONFIG block at the top of VikingsSheetSetup.gs. To add a new team:

  1. Open the sheet → Extensions → Apps Script
  2. Find the teams: [...] array in the CONFIG block
  3. Add a new object following the same format: { name: 'U10', league: 'U10 East Herts League' }
  4. Save and run setupVikingsSheet again — it will add the new team to the Fixtures dropdown and Teams tab
Note: Re-running setupVikingsSheet will clear and rebuild the Fixtures tab headers and dropdowns but will not delete existing fixture rows.

Troubleshooting

ProblemFix
Website shows sample data (not live)Check you're on a deployed domain, not localhost. If deployed, verify the sheet is still Published to web and shared as "Anyone with the link".
Data doesn't appear after a changeWait 1–2 minutes. The gviz endpoint has a short cache. Hard-refresh the browser (Ctrl+Shift+R).
Standings didn't update after a resultUse 🏀 Vikings → Recalculate Standings in the sheet menu to force a recalculation.
Wrong dates showingGoogle Sheets date format works automatically. If pasting dates manually, use DD/MM/YYYY or YYYY-MM-DD format.
Team filter doesn't show a teamThe team may not be in the Teams tab. Re-run setupVikingsSheet after adding it to the CONFIG.
Standings look wrongCheck that team names in Fixtures match exactly (e.g. "U14" not "Under 14"). The script is case-sensitive.
